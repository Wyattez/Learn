
## Container (კონტეინერი)

კონტეინერი არის მსუბუქი, დამოუკიდებელი და შესრულებადი პაკეტი, რომელიც შეიცავს პროგრამის გასაშვებად საჭირო ყველაფერს: კოდს, ბიბლიოთეკებს, სისტემურ ხელსაწყოებსა და პარამეტრებს.

მარტივად რომ ვთქვათ, კონტეინერი არის „მზა გარემო“, სადაც პროგრამა ყველგან ერთნაირად მუშაობს.

---

## Image (სურათი)

სურათი (Image) არის კონტეინერის შაბლონი ან გეგმა, რომლის საფუძველზეც იქმნება კონტეინერი.

ის შეიცავს ყველა საჭირო ინსტრუქციას, ფაილსა და პარამეტრს, რაც საჭიროა კონტეინერის შესაქმნელად და გასაშვებად.

მარტივად რომ ვთქვათ, თუ კონტეინერი არის მზა სახლი, Image არის მისი პროექტი (Blueprint).

---

## Docker Hub

Docker Hub არის ონლაინ პლატფორმა, სადაც ინახება და ნაწილდება Docker-ის სურათები (Images).

ის GitHub-ის მსგავსად მუშაობს, თუმცა კოდის ნაცვლად Docker Images-ს ინახავს.

Docker Hub-დან შესაძლებელია უკვე გამზადებული Images-ის ჩამოტვირთვა და საკუთარი Images-ის ატვირთვა.

---

## Docker Engine

Docker Engine არის Docker-ის ძირითადი კომპონენტი, რომელიც პასუხისმგებელია კონტეინერების შექმნაზე, გაშვებასა და მართვაზე.

ის მუშაობს თქვენს კომპიუტერზე და უზრუნველყოფს, რომ კონტეინერები სწორად გაეშვას და იმუშაოს.

მარტივად რომ ვთქვათ, Docker Engine არის „ძრავა“, რომელიც Docker-ის მთელ სისტემას ამუშავებს.

---

## მოკლე შეჯამება

- **Image** → კონტეინერის შაბლონი.
    
- **Container** → Image-დან შექმნილი და გაშვებული გარემო.
    
- **Docker Hub** → ადგილი, სადაც Images ინახება და ნაწილდება.
    
- **Docker Engine** → სისტემა, რომელიც კონტეინერებს ქმნის და მართავს.
    

### მარტივი სქემა

Docker Hub → Image → Container ← Docker Engine

ანუ:

1. Image-ს ვიღებთ Docker Hub-დან.
    
2. Docker Engine ამ Image-ისგან ქმნის Container-ს.
    
3. Container-ში მუშაობს ჩვენი პროგრამა.




`docker`ეს არის ბრძანება Docker Engine-თან ურთიერთქმედებისთვის.

`run`ეს ქვებრძანება Docker-ს ახალი კონტეინერის შექმნისა და გაშვებისკენ უბიძგებს.

`hello-world`: ეს არის იმ სურათის სახელი, რომლის გაშვებაც გვსურს.

`docker ps` ეს ბრძანება გამოიყენება კონტეინერის სტატუსის შესამოწმებლად

`docker run -d docker/getting-started` -d გამოიყენება კონტეინერმა რომ ფონურ რეჟიმზე იმუშაოს

docker compose -p labspace -f oci://dockersamples/labspace-container-getting-started up -d

docker run -d -p 8080:80 --name my-nginx nginx

| ნაწილი            | მნიშვნელობა                                                                           |
| ----------------- | ------------------------------------------------------------------------------------- |
| `-d`              | **გამოყოფილი** — ფონურ რეჟიმში მუშაობა, ტერმინალის დაბრუნება                          |
| `-p 8080:80`      | **პორტის მიმაგრება** — მასპინძელი პორტის გადამისამართება `8080`კონტეინერის პორტზე`80` |
| `--name my-nginx` | კონტეინერს შემთხვევითი სახელის ნაცვლად, მეგობრული სახელი მიანიჭეთ                     |
| `nginx`           | გამოსაყენებელი სურათი (ავტომატურად ამოღებულია Docker Hub-დან)                         |
კონტეინერის ჟურნალის გამოსავლის სანახავად:
`docker logs my-nginx`

ყოველ ჯერზე, როდესაც ბრაუზერში nginx გვერდს იტვირთავდით, nginx არეგისტრირებდა მოთხოვნას. თქვენ შეგიძლიათ რეალურ დროში თვალყური ადევნოთ ლოგებს შემდეგი გზით:
`docker logs -f my-nginx`

შეგიძლიათ ბრძანება გაუშვათ გაშვებულ კონტეინერში `docker exec`. ინტერაქტიული გარსის გახსნა საჭირო არ არის — შეგიძლიათ ბრძანება პირდაპირ გადასცეთ:
`docker exec my-nginx ls /usr/share/nginx/html`

ეს აჩვენებს nginx-ის მიერ მოწოდებულ ფაილებს. ის, `index.html`რაც ბრაუზერში ახლახან ნახეთ, კონტეინერის ფაილურ სისტემაშია.
`docker exec my-nginx cat /etc/nginx/nginx.conf`

როდესაც კონტეინერს დაასრულებთ, შეაჩერეთ ის:
`docker stop my-nginx`

კონტეინერი გაჩერების შემდეგაც არსებობს — ის უბრალოდ აღარ მუშაობს. შეამოწმეთ:
`docker ps -a`

ახლა შექმენით თქვენი სურათი. დარწმუნდით, რომ იმ დირექტორიაში ხართ `project/`, სადაც თქვენი `Dockerfile`საცხოვრებელი ადგილია:
`docker build -t getting-started .`

შედით Docker Hub-ში:
`docker login`

მონიშნეთ თქვენი ლოკალური სურათი თქვენი Docker Hub მომხმარებლის სახელით, რათა Docker-მა იცოდეს, სად განათავსოს იგი:
`docker tag getting-started wyattez/getting-started`

დააჭირეთ:
`docker push wyattez/getting-started`

თქვენი სურათი ახლა Docker Hub-ზეა. ნებისმიერს შეუძლია მისი ამოღება და გაშვება:
`docker run -p 3000:3000 wyattez/getting-started`



# 🐳 Docker – მოკლე დოკუმენტაცია (Containers & Images)

## 📌 1. Container-ის ნახვა

### მხოლოდ გაშვებულები:

```
docker ps
```

### ყველა (running + stopped):

```
docker ps -a
```

---

## 📌 2. Container-ის გაჩერება

```
docker stop <container_name_or_id>
```

მაგ:

```
docker stop nginx-1
```

---

## 📌 3. Container-ის წაშლა

```
docker rm <container_id_or_name>
```

### ყველა container-ის წაშლა:

```
docker rm -f $(docker ps -aq)
```

---

## 📌 4. Image-ის ნახვა

```
docker images
```

---

## 📌 5. Image-ის pull (ჩამოტვირთვა)

```
docker pull nginx:1.14-alpine
```

👉 მხოლოდ ჩამოტვირთავს, container არ იქმნება

---

## 📌 6. Image-ის წაშლა

```
docker rmi <image_name>
```

### ყველა image-ის წაშლა:

```
docker rmi -f $(docker images -q)
```

---

## 📌 7. Container-ის და Image-ის განსხვავება

- **Image** → template (შაბლონი)
- **Container** → გაშვებული instance (რეალური პროცესი)

---

## 📌 8. Cleanup (სრული გასუფთავება)

```
docker rm -f $(docker ps -aq)docker rmi -f $(docker images -q)
```

---

## 📌 9. მდგომარეობები (States)

- **Running** → მუშაობს
- **Exited** → გაჩერებულია
- **Created** → შექმნილია, მაგრამ არ არის გაშვებული

---

## 📌 10. გადამოწმება

```
docker ps -adocker images
```
